Stage 1 - Define layout

In the HomePage, we have clearly two sections, one is the Header (Nav) and the other one is the directory that allows user to click and start browsing right away.

We can choose one to start with, or create basic layouts for both at the same time.

For simplicity, we are starting with the directory section this time.

Stage 2 - Build HTML and css for the layout

A. 3 things set up in advanced, index.js connects to app.js and app.js connects to homepage.component.js.

B. Now we can build the basic HTML and css here for the homepage (will add nav component in the future) which helps ourselfs to clarify what are we going to build and what components do we need.

Apparently, we need a directory-menu to contain all five menu-item, and we have title and subtitle inside each menu-item.

Stage 3 - Component Architecture

We now know what are we going to build.
We need to destructure the html into components.

There are two main types of folder we need, which are components (reusable) and pages (non reusable).

1. HomePage component obviously will be used once only, so it belongs to the page folder.

2. directory component and menu-item component which are inside of the HomePage component will probably be reused, so they will be in the component folder.

Stage 4 - Replace HTML and css with components

We just need to easily copy and paste the HTML (now jsx) and scss code to the specfic component.

Of cause we need to build the hierarchy correctly (directory has menuItem)

NOTE:

1. The directory component is a class component contains a state, because we want to populate data from an array of objects.
