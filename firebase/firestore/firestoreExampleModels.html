Votes

We want to record the vote for the users and the posts, thats why we have upvotes field in each user / post doc.

In the votes collection, it acts as the middle man collection that the document id will be consisted with user doc id and post doc id.

The votes collection doc contains fields about the user doc and post doc.

The total vote can be calculated using Cloud Function.

My Suggestion

Consideration 1: We are using enormalization din the votes collection, although the duplicated data (userid and postid) will not be changed frequently, changing them can cause writing to lots of document. (depends on the related doc)

Solution 1: Changing the userid and postid into some unique identifier that will never change.
