React

What is react?

React is a very strong javascript library created by Facebook to solves the problems that they had in the past building large project.

Before React

Before we had React, in the 90s, we only have Html, css, and Javascript

1. Jquery came up

Jquery = Easier to interact with the DOM + Unified syntax across different browsers (compatibility)

2. Backbone js came up due to the websites getting bigger and harder to organise the javascript files

Then it was easier for us to manipulate the DOM, so single page application born. More focus on Javascript, less on HTML (Javascript change the HTML)

3. Angular js came up in 2010 allows people to better organise the javascript files.

But the things getting more complex, more interactive things are built on the website... Clicking one place, will trigger event... DATA FLOWS EVERYWHERE

4. React came up in 2013 to solve problems, it brought us a lots of new development theories. Imperative -> Declarative

Why do we need React?

Before we had react, we manipulate the DOM by using querySelector, getElementById and addEventListener or we might also use jquery to help us to write code.

However, if the project gets more complex, the problems come up.

  • Inconsistency
  • Hard to scale
  • Hard to manage
  • unreusable
  • Slow development

React solves the problems!

Using react is like we are throwing the ingredients (components) and it does all the magic underneath the hood to push up the website.

React has View to make the DOM changes in a predictable way in order to be scalable and managable.

React also has React Bot and Virtual DOM which we only need to describe things in to the virtual dom (render() in JSX) and the react bot will optimise the code and apply it to the actual DOM for us.

Think In Components

1. We first create a javascript file for a component that you want to add into the ReactDOM.

2. Then import React.Component using Javascript ES6 Module.

3. After that, we can build a class for the component and specify how do you want to render using the render().

We use JSX, a syntax that looks alike HTML to describe what do we try to render.

We can import a css file specifically for the component js

4. After creating the component, we can now use it everywhere (reusable). We can put it into the index.js to the ReactDOM using the self closing tag.
