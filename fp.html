Functional Programming

What is functional programming?

Functional Programming features

The idea of Functional Programming is that the data and the function should be separated. (seperation of concerns)

So that when we debug our code, we don't need to debug this part of the code but aim for others, because they are built with pure functions.

Functional Programming suggests that the function should be pure, so that the program can become predictable and also heigher testability.

It minimizes the side effect of a function makes in order to increase the readability and decrease the bugs that can be triggered.

Bad practice in FC: Different functions handle the same piece of data at the same time...

What is pure functions?

No side effects

Pure function should not modify anything outside the function itselfs.

It suggests that function must return something (since it is not mutating anything outside). Function without returning something is just a procedure. 

            
                const array = [1, 2, 3]; //An array defined outside of the pure function.

                function pureFunc() {
                    const pureFuncArray = [...array]; // Deep copy the outsider so that we don't need to modify it directly

                    pureFuncArray.pop();

                    return pureFuncArray; // Pure Function always return something
                }

Pure function has to return return something since it can not modify outsiders. 

                
                    function notPure() {
                        console.log('I am not a pure function');
                    }

It is not a pure function, because it use console.log to commmuniate with the outsiders for displaying text.

Same input -> Same output (Referential Transparency)

With the same input, pure functions always return the same output. 

                
                    function sum(num1, num2) {
                        return num1 + num2;
                    }

                    sum(2, 5); // It always return 7

                    /////////////////////////////////////

                    function timesFive(num) {
                        return num * 5;
                    }

                    timesFive(sum(2, 5)) // Always return 35

                    //Referential Transparency
                    timesFive(7) // Even we switch the sum function to 7 explicitly, the answer will be the same.

Referential Transparency indicates the same input will generate the same output, so we can predict the answer. Even we explicitly use the predicted answer to other function, the output of the other function will be the same.

What is Idempotence?

It is like the same input -> same output in pure function

Idempotence function doesn't have to be a pure function, but it indicates that the same request (API Request) / function call being made multiple times will not change the result. This provides us a predictable result.

                    
                        function log(num) {
                            console.log(num);
                        }

                        log(5);
                        log(5);
                        log(5);
                        log(5);
                        log(5); //Always return 5

The above example is not a pure function, but it is idempotence because it generate the same output no matter how many times we call the function. Just like the API HTTP method (get, delete, put).
