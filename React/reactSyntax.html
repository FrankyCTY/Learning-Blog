React Syntax

In this page, I mark down the syntax or things that I want to memorize better.

Props object

Props object allows us to pass data to the component (pure function) similar to a regular function's argument.

Props is just an object, as everything in javascript are objects.

We can see that we have passed testProps to the component, and we can call it by using 'props.testProps'. If we use deconstructor {testProps}, we can directly use {testProps} in the JSX.

Props children

As you can see on the above and the image on the right, the props object has a property called 'children'.

It allows us to access the children component that is wrapped by the current component.

The current component (Scroll) wraps a child component (CardList).

So now in the Scroll component page, we can describe that the CardList (props.children) is wrapped by a div with scroll feature. (image on the left)

Asynchronous setState

The setState() method is asynchronous, and react might group different setState calls together as a single update.

Left img: The console log is after the asynchronous setState(), so it will be ran before the 'meaningOfLife' state modified.

Right img: setState() actually accepts a call back function as the second argument, but it is considered as a bad practice to use setState() method to modify the state's property.

WHY?

Because since react might group different setState calls together as a single update. It can lead to unexpected result as the state's property can be shared by other setState call.

prevState

In the asynchronous setState part, we have talked about the problem of changing state's property using object as an argument in the setState() method. But we haven't talk about how to solve it.

The first argument of the setState() method actually accept a function, and this function receives two arguments: prevState and prevProps. 100By playing with the prevState, we can prevent the share state gets messed up.
