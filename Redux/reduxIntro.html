Redux

Why do we need Redux?

⭐We use this.state on the class components to achieve the one way data flow (state driven), but states are getting harder to manage once the application gets bigger.

⭐In a complex application, a state needed to be passed very deeply for a child (prop drilling), which makes the code unmaintainable and unreadable.

What is Redux?

We can use Redux to help manging the state.

⭐Redux is a big javascript object that store all the states and used to describe how our application should look.

We only need to pass down the state to whichever component needs it as its props.

Struggling like how to manage states and how to modify child component's state as a parent component are now gone/ .

⭐Redux also seperate the UI and the client side calculations to the Redux folders which we might had to put in the component file without Redux.

Redux Concepts

Redux is good at:

  • Managine Large State
  • Useful for sharing data between components
  • Predictable state management using the 3 principles

The 3 principles:

  • Single source of truth (The big javascript object)
  • State is read only (immutability)
  • Changes using pure functions (predictable code / less error prone)

Single source of truth: We store states inside of the big javascript object for easier state mangement.

State is read only: We make the state to be immutable so that our application will have less error. Each actions taken by the user will create new state instead of modifing.

Changes using pure functions: Again, it makes the state immutable and predictable.

Redux Flow

  • Action.js - An file contains info about the user actions, containing the definition of different actions.
  • Splited Reducers - Hold its own initial state, and connect the prevState with the action. (pure function)
  • Root Reducer - A final reducer that combines the splited reducers and being imported to the store file.
  • Store - The entire state of the application and let us to add middleWare.
  • DOM changes - State changes lead to react bot to make DOM changes

As you can see, the Redux flow is also a one way data flow.
