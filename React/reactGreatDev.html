Great React Developer

How to be a great React Developer?

To be a great React Developer, we need to good at 3 things.

  • Where to put state?
  • How to define a component?
  • What changes due to a specific state changes?

1. Where to put state?

Remember react adopts the one way data flow which makes the code more predictable and easier to be debugged.

In react, we can pass state to the children components as the props, and the children components can react to the state changes.

OR

We can trigger changes to the state to re render the application by clicking on a log in button.

So we need to put the state on a position that can cover the components that it will implicate, but not too far away to cover unrelated components for better organization.

2. How to define a component?

In this example, we build a component for "cardList" and it will return a list of card components.

Why do we build a component for card list?

For reusability, we can create another list whenever we want by changing the props that passed into this component.

3. What changes due to a specific state changes?

Understand what the state is about, and what components it will relate to.
